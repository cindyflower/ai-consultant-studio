Apps Script 安裝與設定
1
開啟 Apps Script 編輯器
在 Google Spreadsheet 點上方選單「延伸功能」→「Apps Script」
2
貼上程式碼
把編輯器裡的預設內容全部清除，貼上提供的 Apps Script 程式碼
3
確認模板 ID
程式碼第一行有 TEMPLATE_ID，確認已填入正確的 Google Slides 模板 ID（從模板網址取得）
Apps Script
// 從模板網址取得 ID：
// https://docs.google.com/presentation/d/【這裡就是ID】/edit
const TEMPLATE_ID = '13RrTgv8Ptr2b_EmjG0iWazh8vmji04UEjU1UwrPgCgM';
4
儲存並重新整理
按 Ctrl+S 儲存，關閉 Apps Script 頁籤，回到 Spreadsheet 重新整理（F5），上方會出現「📊 產生周報」選單
5
第一次授權（只需做一次）
執行時 Google 會跳出安全警告，這是正常的。點「Advanced」→「Go to... (unsafe)」→「Allow」完成授權，之後不會再跳出。
⚠
為什麼會出現警告？因為這是你自己寫的程式，不是 Google 官方上架的應用。授權對象是你自己的 Google 帳號，完全安全。

每週操作流程
✓
設定完成後，每週只需要以下 2 個步驟即可產出完整周報。
Step 1 — 填寫資料
團隊成員照常在 Google Spreadsheet 填入當週資料，Hardware 填 Hardware 工作表，Software 填 Software 工作表。
ℹ
Progress / Plan / Problem 欄位，每個項目用「- 」（減號 + 空格）開頭，程式會自動轉成條列格式。
Step 2 — 產生 Slides
點 Spreadsheet 上方選單「📊 產生周報」→「產生本週 Slides」，等幾秒鐘，畫面會彈出今天日期命名的新 Slides 連結。
✓
每次產出都是全新獨立檔案，不會覆蓋上週的周報，可在 Google Drive 找到。
⑦
常見問題排除
❌ 錯誤：Source page could not be found
模板 ID 設定錯誤，或模板分享權限不足。請確認 TEMPLATE_ID 正確，並確認模板已設為「知道連結的人可以編輯」。
❌ 錯誤：The object has no text
模板裡有空白的文字框。請確認所有預期有內容的欄位都已填入對應的 {{標籤}}，不能留空白。
❌ 標籤沒有被替換（投影片還顯示 {{HW_STO}}）
Spreadsheet 的工作表名稱必須是 Hardware 和 Software（大小寫須一致）。或檢查欄位順序是否與程式設定的一致。
❌ Health Status 色塊沒有變色
Health 欄位的值必須完全符合五種狀態之一：BLOCKED / AT-RISK / ON-TRACK / COMPLETE / PLANNING（全大寫，包含中間的連字號）。建議使用下拉選單避免拼字錯誤。
